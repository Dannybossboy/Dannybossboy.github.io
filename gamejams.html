    The 55th Expedition

    Project Details

    • Game Type: Game Jam
    • Engine: Unity
    • Development Time: 1 week
    • Team-size: 1
    • Genre: 3D, Dungeon Crawler, adventure, turn based
    • Status: Finished

    Overview

    The idea for this game was to make a 3D dungeon crawler where the player has to navigate a maze filled with enemies and reach the end to defeat the dragon. The player gains xp from defeating enemies, gathers materials to sell and uses equipment to do and take more damage. This ended up being one of my favorite games Ive worked on and one of my most well recieved games.

    My Contributions

    I was the sole developer of this project.

    Level Design

    • Created a level using the custom editor to encourage exploration and guide the player towards the exit.

    Art

    • Created all the 3D art, the 2D art was taken from 3rd party sources.

    Programming

    • Created a basic 3D grid based movement controller. Similar to classic dungeon crawler movement.
    • Designed a custom level editor to simplify the level creation process. This worked by taking in a map layout image and converting that to a basic 3D map.
    • Developed a complex drag and drop inventory system, allows for features such as using items, switching items and viewing details on hover.
    • Created a player leveling system that allows the player to gain exp, level up, allocate points into attributes and witness the effects of doing certain builds.
    • Created a complex turn based combat system that allows players to attack, use items, run or even talk.
    • Used the modular inventory system to add a simple shop NPC that allows selling and buying items.

    Beneath the Mountains

    • Engine: Unity

    • Team-size: 1
    • Genre: 3D, Adventure, metroidvania
    • Status: Finished

    Mcboom's Kaboom

    Project Details

    • Game Type: Game Jam
    • Engine: Unity
    • Development Time: 1 week
    • Team-size: 4
    • Genre: 2D, Puzzle, Platformer
    • Status: Finished

    Overview

    This project was part of my university's Game Dev Club's first game jam of the semester. I worked with an extremely talented team, we started with a few concepts and it snowballed until we had an interesting concept and mechanics. The idea was that you play as a villian who planted a bomb in a tower and you have to escape before it goes off. You have to use your time rewind mechanic to solve puzzles and make it out before 1 minute passes.

    My Contributions

    Besides team mangement and project planning. Heres the list of my Contributions.

    Level Design

    • Worked off concepts provided by the art team to alter and construct levels.

    Programming

    • Time rewind system: Used abstract classes to give objects unique behaviors during time rewind. As well as replaying positions and rotations physics objects made over time.
    • Player Movement: Created a intermediate player controller featuring jump-cuts, coyote time and altered gravity make jumps feel good.
    • 2D Physics Objects: Implemented satisfiyng physics based movement of objects such as cogs turning, pistons extending and boxs falling. All which inherit methods that controls behavior during time rewind.

    The Last Shroom

    Project Details

    • Game Type: Game Jam
    • Engine: Unity
    • Development Time: 2 weeks
    • Team-size: 4
    • Genre: 2D, Rougelike, arcade
    • Status: Finished

    For this project I worked with 3 talented team members I found on the jam website, 2 artist, 1 coder(me) and 1 sound designer.     • Crafted a simple and satisfiyng combat system for players to use against enemies.
    • Developed simple enemy AI that gives unique behavior for each enemy.

    The Knight's Rescue

    Project Details

    • Game Type: Game Jam
    • Engine: Unity
    • Development Time: 3 days
    • Team-size: 1
    • Genre: 2D, Puzzle, Platformer
    • Status: Finished

    Overview

    This project was part of my university's Game Dev Club's first game jam of the semester. I started off the project with 1 other team member, but due to him being busy I ended up being the sole developer of this game. The idea was to create a short platformer with the basic mechanic of toggling lava and water to overcome small puzzles.

    My Contributions

    Level Design

    • Created each level with a simple and clear goal, while progressively making each level more difficult.

    Programming

    • Created the switch mechanic which turns off and on sections of water which you can swim in and lava which kills you.
    • Player Movement: Created an advanced player controller featuring jump-cuts, coyote time, jump buffering, air gravity/friction and satisfiyng squash and stretch to add juice.
    • Coded the rest of the miscellinious features such as UI navigation, respawning, level switching and cutscenes.
